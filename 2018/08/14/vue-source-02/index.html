Vue源码解读 02 - vm.$mount, mountComponent, updateComponent

2018年8月14日

上回讲到new Vue后, vm实例进行了一系列初始化并代理了data, methods, props, 还执行了beforeCreate与created生命周期, 现在要执行vm.$mount将实例挂载到页面上.

vm.$mount

我们先看看$mount的定义

首先要明确一个概念: Vue在mount阶段只认render函数.

一般项目开发时, 会先经过webpack等工具对.vue文件进行编译打包, vue-loader会将.vue文件中的template编译为render函数.

在这种普遍场景下, 直接执行src/platforms/web/runtime/index.js中的$mount方法, 获取el对应的dom元素, 执行mountComponent方法

但如果没有用单文件组件的形式开发, 也没有在vue的配置对象中写render函数, Vue则会通过一系列操作确保有render函数的存在(src/platfroms/web/entry-runtime-with-compiler.js), 它们包括:

  1. 如果定义了template, 则获取对应的template的innterHTML
  2. 如果没有定义template但定义了el, 则获取el的outerHTML
  3. 获取1或2中得到的html字符串, 在运行时将其编译为render函数并挂载给vm.$options
  4. 仍然调用行src/platforms/web/runtime/index.js中的$mount方法, 获取el对应的dom元素, 执行mountComponent方法

mountComponent

mountComponent方法的流程:

  1. 缓存el为vm.$el
  2. 再次检测render函数是否存在, 如果有问题则抛警告
  3. 执行生命周期beforeMount
  4. 定义updateComponent方法, 内部调用vm._update(vm._render()), 其中_render方法负责生成vnode, _update方法负责将vnode的改变或新建更新在页面上.
  5. 创建一个renderWatcher, 关联updateComponent方法
  6. 如果该vm实例的虚拟父vnode为空, 证明该节点是根节点, 将vm._isMounted设置为true
  7. 执行生命周期mounted
  8. 返回vm实例

RenderWatcher实例化过程

mountComponent的new Watcher过程创建了一个渲染watcher, 大致流程如下:

  1. 将渲染watcher赋值给vm._watcher, 并push入vm._watchers数组.
  2. 初始化
  3. 将updateComponent方法赋值给watcher.getter
  4. 执行watcher.get方法 -> watcher.get方法执行this.getter方法 -> 实际执行updateComponent方法 -> 执行vm._update(vm._render())

再次强调, vm._render方法会返回vnode, vm._update会将vnode挂载到页面上.

总结

总结下, $mount方法会先编译出render函数并挂载给vm.$options, 再调用mountComponent方法, 创建一个渲染watcher, 其在初始化的过程中, 会调用一次updateComponent方法, 其中调用vm._update将vm._render生成的vnode挂载在页面上.

下节将继续向下分析vnode的生成和挂载过程.
