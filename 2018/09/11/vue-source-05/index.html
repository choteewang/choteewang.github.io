Vue源码解读 05 - update, 将vnode渲染到页面dom上

2018年9月11日

上节讲到了vm._render方法返回了vnode, 并即将调用vm._update方法将vnode渲染到页面上.

vm._update定义在lifecycleMixin中, 也是在大Vue构造函数导出的过程中挂载的.

首次渲染prevVnode肯定为空, 所以实际上调用的是vm.$el = vm.__patch__(vm.$el, vnode, hydrating, false).

__patch__方法也是在Vue导出过程中挂载的方法, 它实际是createPatchFunction方法的返回值. 其中nodeOps封装了在当前平台下(web)操作dom的方法, const modules = platformModules.concata(baseModules)是当前平台操作html特性过程方法的封装.

将nodeOps, modules传递给createPatchFunction后, 会先装载snabdom的生命周期钩子, 并针对当前平台进行一系列初始化和抹平操作, 最后return patch方法.

这里作者运用了函数科里化, 让最后return的patch function 向上寻找的createPatchFunction闭包作用域具有当前平台下将vnode挂载在真实dom节点上的各种方法, 但又不用担心平台差异(平台差异已经被闭包部分抹平封装好).

patch方法的第一个参数oldVnode在首次渲染时是真实dom(vm.$el), 首次渲染时isRealElement为true, 然后会调用oldVnode = emptyNodeAt(oldVnode)

这里会把vm.$el转化为vnode

然后记录当前元素的真实dom和其父元素的真实dom. 然后调用createElm方法.

createElm方法的流程:

  1. 首次渲染时会先生成vnode.elm, 用dom原生api生成一个dom element,
  2. 执行createChildren, 为当前vnode的所有子节点vnode生成dom并插入到vnode.elm中
  3. 执行insert方法, 根据父节点和参考节点(首次渲染刚才传的是下一个兄弟元素)将当前当前vnode对应的dom元素插入到页面中.

createChildren方法做的事: 对当前vnode的children递归调用了createElm.

可以分析出, children的dom element是被先插入到当前元素的 dom element 中, 最后当前元素的 dom element 才会被插入到页面上

总结

01-05, 我们分析了:

提供render函数, 且render函数内部是渲染一个html保留元素, 从new Vue()到元素挂载到页面上的过程

总结下过程是:

  1. 初始化生命周期, 事件, render函数, 代理data, methods, 期间执行beforeCreate和created生命周期
  2. 执行vm.$mount, 这个阶段只认render函数, 如果没有render函数, 执行运行时编译过程生成render函数
  3. 执行mountComponent方法, 传入vm实例, 将el对应的dom元素挂载在vm.$el上, 定义updateComponent方法, 内部调用vm._update(vm._render()).
  4. 创建一个render Watcher, 将updateComponent方法赋值给watcher.getter, 执行watcher.getter方法, 实际上执行了vm._update(vm._render())
  5. vm._render方法执行, 用户书写的render函数被执行, 并将vm.$createElement方法作为参数传递给用户,
  6. vm.$createElement方法内部调用createElement方法, 执行normalizeChildren过程(这个过程如果children中由的值还是一个children数组, 则递归调用normallizeChildren), 将children拍平为一维vnode数组(children中如果有子节点是$createElement返回的, 会形成递归调用$createElement, 返回子节点vnode), 最后, 创建当前节点的vnode并返回, 这个返回值也是render函数返回值
  7. vm._update方法接到vm._render的返回的vnode, 调用由createPatchFunction方法抹平平台差异后的返回值__patch__函数, 该函数内部通过递归调用createElm方法将所有节点的vnode创建dom element并插入到页面中(顺序是先将子节点的dom element插入到父节点的dom element中, 再将父亲元素的dom element插入到上层父节点或页面中)

至此, 我们详细的分析了html保留元素的初始挂载过程, 后面的章节, 将关注自定义组件的挂载过程.
